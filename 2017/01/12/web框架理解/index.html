web框架理解

  • framework
  • spring

前面一段时间把黄勇的书看了一遍，自己敲了一个简单的web框架
记录一下这个web framework框架的执行流程，WebFramework简称Web框架,比较知名的比如Spring,
其主要是通过技术手段管理模块中的类的生命周期,更加有效的管理内存

入口

首先，整个框架的入口放在DispatcherServlet
重写Servlet的init函数用来初始化，通过类加载工具，HelpLoad初始化一系列的类,

通过HelperLoader进行循环加载

在整个框架当中这几个Help类的作用至关重要,ClassHelp主要用来加载让框架管理的Class,
BeanHelp主要用来实例化Class(IOC控制反转),DIHelp(用来进行依赖注入ps:名字改了，原文具体啥名字我也不记得了)
ControllerHelp是用来加载Controller类的

基本上每个Help类都有两部分，

  • 静态容量：一个静态变量用于存放静态代码获取到的Object
  • 静态块：保证在类加载后静态代码被马上执行,并且把数据存放到静态成员变量

加载类

加载类主要用到ClassHelp,

  • 首先获取当前线程的类加载器(Thread.currentThread().getContextClassLoader())
  • 扫描文件夹/Jar包/.class文件取出加载了框架的所有Class并放入CLASS_SET

IOC控制反转

  • 通过BeanHelper获取到所有的Class
  • 通过反射工具实例化所有Class
  • 把Class作为键，实例化的对象作为值存入Map里面,这个。。就是著名的IOC了…

依赖注入DI

看了一下我朋友写的blog，他说黄勇先生在一开始就混淆了依赖注入和控制反转的概念
我仔细的想了一下，确实这是两个概念…(也是纯属于个人见解..)

  • 遍历beanMap取出key和值
  • 取出key里面的所有字段
  • 判断字段是否打了Inject注解
  • 如果有Inject注解，通过Field.getType()取到Filed的Class
  • 通过Filed的Class去map里面寻找对应实例化好的对象
  • 通过反射工具注入到Field字段

依赖注入其实就是通过反射Filed.set()方法在框架加载类的时候进行判断注入,
由于控制反转(IOC)经常和依赖注入(DI)经常配合使用,
所以好多将两者混为一谈,其实并不是这样.IOC是一种去耦合手段,而DI则是链接耦合的手段

请求映射

  • 先获取所有Controller,然后获取Controller里面的Action注解的方法
  • 获取方法的value（请求路径）以及method（请求方法get,post…）组成一个Request对象
  • 获取方法的名字以及所在的Class组成一个Hanlder对象
  • 以Request为key,Handler为值存入一个静态map,方便接收请求的时候与map里面的值做判断,通过反射调用Controller的方法…

总结

  • 框架通过反转控制和依赖注入保障单例,节省了内存
  • 框架定义好了Bean,使用这些Bean规范化了参数和返回的Object
